🔜 MODIFICACIONES PENDIENTES (definidas, por implementar)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. SISTEMA DE INVITACIONES POR ROL
   Problema: no hay forma de agregar usuarios sin acceso técnico a la base de datos.
   Solución:
   · El super admin genera un link de invitación con rol y empresa pre-configurados.
   · El invitado abre el link, completa nombre/apellido/email/contraseña y queda registrado.
   · El super admin gestiona usuarios desde un panel: ver, agregar, eliminar.
   Estado: diseñado, pendiente de desarrollo.

2. REDISEÑO DE CARD TOP PERFORMER
   Problema: el número del índice como protagonista es frío y poco informativo.
   Solución:
   · El protagonista pasan a ser las fortalezas cualitativas derivadas de las métricas:
     "Validación perfecta" / "Consistencia excepcional" / "Alta evidencia" / "Autoevaluación precisa"
   · El número del índice queda como dato secundario en la tabla de ranking.
   Estado: concepto definido, pendiente de desarrollo.

3. ÍNDICE TRAZA AUTÓNOMO (doble índice)
   Problema: el desempeño de un empleado no puede depender solo del criterio
   de un supervisor. Un supervisor injusto, sesgado o ausente distorsiona el índice.
   Solución: sistema de doble índice paralelo.

   TRAZA Validado (60%) — el módulo actual con validaciones humanas.

   TRAZA Autónomo (40%) — calculado 100% a partir de comportamiento observable,
   sin que ninguna persona pueda influirlo. Mide 5 señales:
     · Consistencia: regularidad con que registra avances
     · Densidad de evidencia: % de avances con prueba adjunta
     · Proactividad: si actualiza por iniciativa propia o esperando al supervisor
     · Precisión histórica: alineación entre autoevaluación y validación final a lo largo del tiempo
     · Progresión: si sus objetivos crecen en ambición y complejidad

   Protección automática: si TRAZA Autónomo >> TRAZA Validado, la plataforma
   genera una alerta para el super admin (posible sesgo del supervisor).

   Nota técnica: no requiere API externa. Se calcula con datos ya en la plataforma.
   Estado: concepto definido y documentado, pendiente de desarrollo.

4. NARRATIVA AUTOMÁTICA CON IA (Anthropic API)
   Problema: el índice es un número; los empleados necesitan contexto para entenderlo.
   Solución: usar la API de Anthropic para generar un párrafo personalizado
   por empleado que explique su índice en lenguaje natural.
   Ejemplo: "Tu consistencia es alta y tu evidencia está bien documentada.
   El principal foco de mejora es la alineación con tu supervisor, donde
   existe una brecha de 18 puntos en el último trimestre."
   Este texto aparecería en la credencial y en el perfil del empleado.
   Estado: diseñado, requiere activar clave de API de Anthropic.

5. PLATAFORMA MULTI-EMPRESA
   Problema: TRAZA funciona para una empresa pero no está preparada para escalar.
   Solución: permitir que cualquier empresa se registre con su propio espacio,
   usuarios y datos completamente aislados.
   Cada empresa tendría:
   · Su propio super admin
   · Sus supervisores y empleados
   · Objetivos, avances y métricas aislados por empresa
   · Posibilidad de configurar el peso de los módulos del índice
   Nota: la arquitectura de empresa_id ya está en la base de datos.
   El siguiente paso es el sistema de onboarding de nuevas empresas.
   Estado: arquitectura lista en DB, falta el flujo de registro y onboarding.
