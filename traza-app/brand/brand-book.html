Brand Story

Las organizaciones llevan décadas evaluando personas con herramientas de otra época: formularios anuales, percepciones subjetivas, conversaciones que no dejan registro.


TRAZA nació para cambiar eso.


Creemos que cada persona merece claridad sobre lo que se espera de ella, y transparencia sobre cómo se va a evaluar su trabajo. Cada objetivo registrado en TRAZA es una promesa documentada. Cada avance, una pieza de evidencia. Cada validación, un reconocimiento objetivo.


No controlamos a las personas: les damos las herramientas para que su trabajo hable por sí mismo.

03

Logo & Branding

Concepto del logo: El isotipo es una "Z" geométrica (de traZa) construida con tres elementos precisos: barra superior, diagonal y barra inferior. La diagonal representa el acto de trazar — el registro continuo. La forma es limpia, arquitectónica y memorable. No usa metáforas literales (sin huellas, caminos ni flechas genéricas).

Área de Seguridad

El área de seguridad del logo equivale a la altura de la letra "a" del wordmark en todos sus lados. Nunca colocar otros elementos visuales dentro de esta zona. En el isotipo, el área de seguridad es igual a 0.25x el alto del ícono por cada lado.

08

Principios UX

⬜
Espacio sobre densidad
Padding mínimo de 20px en cards. Nunca comprimir información. El espacio es claridad visual.
🧠
Baja carga cognitiva
Mostrar solo lo necesario en cada paso. Un máximo de 3 acciones primarias por pantalla.
📊
Información progresiva
Lo más importante primero. Los detalles se revelan cuando el usuario los necesita.
✅
Feedback inmediato
Cada acción del usuario debe tener respuesta visual en menos de 100ms.
🎯
Jerarquía clara
Una sola acción primaria por sección. La jerarquía visual guía el recorrido del ojo.
♿
Accesibilidad
Contraste mínimo AA (4.5:1 para texto). Focus visible siempre. ARIA labels en íconos.

Regla de oro TRAZA: Si un elemento no tiene una función específica, no existe. Cada píxel en pantalla debe justificarse con un propósito claro para el usuario.

Manual de Marca — Reglas de Uso

✅ Siempre hacer
  • ✓ Usar "traza" en minúscula en el wordmark y la UI.
  • ✓ Mantener el ratio del isotipo. Nunca deformarlo.
  • ✓ Respetar el área de seguridad del logo.
  • ✓ Usar los tokens definidos. Nunca valores arbitrarios.
  • ✓ Priorizar espacio en blanco sobre densidad.
  • ✓ Usar Inter para UI y Plus Jakarta Sans para headings.
  • ✓ Verificar contraste mínimo AA en textos.
❌ Nunca hacer
  • ✕ Capitalizar "Traza" o "TRAZA" en el logo.
  • ✕ Usar colores fuera de la paleta aprobada.
  • ✕ Agregar gradientes al logo.
  • ✕ Usar el isotipo con fondo de color que no sea primario, negro o blanco.
  • ✕ Usar tipografía con serif en ningún contexto de producto.
  • ✕ Mezclar dos bibliotecas de iconos.
  • ✕ Colocar el logo sobre fondos con bajo contraste.

Definición de "producto de primer nivel": No es la cantidad de elementos en pantalla. Es la coherencia absoluta entre cada pixel y el sistema que lo define. Si hay una duda sobre qué usar, la respuesta siempre está en este documento.
